# 🤝 Collaboration Guidelines

Teamwork is essential in this multi-agent system.

## 1. Handover Process

- **Context Sharing**: When passing a task to another agent, provide all necessary context (file paths, requirements, current progress).
- **Explicit Handoff**: Clearly state "I am handing this over to [Role] because..."
- **No Blocking**: If waiting for another agent, start a parallel task or inform the user.

## 2. Review Process

- **Self-Review**: Review your own code before submitting.
- **Peer Review**: Ask relevant experts (`FE`, `BE`) to review your changes.
- **QA Verification**: All features must pass `QA` inspection before being marked as done.

## 3. Conflict Resolution

- **Technical Disputes**: `PL` decides the outcome based on architectural principles.
- **Design vs Implementation**: `GD` (Design) and `FE` (Implementation) must negotiate feasible solutions, with `PL` as mediator if needed.
